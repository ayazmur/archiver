# Python Archiver with Password DB

Простой консольный архиватор на Python, который умеет создавать и распаковывать ZIP-архивы. Его главная особенность — встроенная база данных паролей. При распаковке защищенного архива он сначала пытается использовать пароли из этой базы.

## Возможности

*   Создание ZIP-архивов из файлов или папок.
*   Защита архивов паролем.
*   Ведение локальной базы данных (`SQLite`) часто используемых паролей.
*   Автоматический подбор пароля из базы при распаковке.
*   Если пароль из базы не подошел, запрашивает его у пользователя.
*   Возможность сохранить новый, успешно использованный пароль в базу.

## Установка и настройка

1.  Клонируйте репозиторий:
    ```bash
    git clone <URL вашего репозитория>
    cd <название папки репозитория>
    ```

2.  Программа не требует внешних библиотек и работает на стандартном Python 3.

При первом запуске любой команды будет автоматически создан файл базы данных `passwords.db`.

## Использование

Все команды выполняются через `archiver.py`.

### Управление паролями

*   **Добавить пароль в базу:**
    ```bash
    python archiver.py pass add "my_secret_password"
    ```

*   **Посмотреть все пароли:**
    ```bash
    python archiver.py pass list
    ```

*   **Удалить пароль:**
    ```bash
    python archiver.py pass del "my_secret_password"
    ```

### Создание архива

*   **Создать архив без пароля:**
    ```bash
    # Создать my_docs.zip из папки 'documents'
    python archiver.py create documents my_docs.zip
    ```

*   **Создать архив с паролем** (пароль будет автоматически добавлен в базу):
    ```bash
    python archiver.py create secret_file.txt secret.zip -p "Super-Secret-123"
    ```

### Распаковка архива

*   **Распаковать архив:**
    Скрипт сам определит, нужен ли пароль, и попробует подобрать его из базы.
    ```bash
    # Распаковать в текущую папку
    python archiver.py extract secret.zip
    ```

*   **Распаковать в указанную папку:**
    ```bash
    python archiver.py extract secret.zip -d unpacked_files
    ```

## CI (Непрерывная интеграция)

Этот репозиторий настроен с использованием GitHub Actions. При каждом `push` или `pull request` в ветку `main` запускается автоматическая проверка:
1.  Код проверяется на стилистические ошибки и простые баги с помощью `flake8`.
2.  Выполняется "дымовой тест" (`smoke test`), который проверяет, что скрипт в принципе запускается.

Это гарантирует базовое качество и работоспособность кода.
